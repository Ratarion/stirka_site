# Сайт для администрирования Telegram‑бота записи на стирку

**Веб‑панель управления** для мониторинга и администрирования Telegram‑бота, автоматизирующего запись на стиральные и сушильные машины в студенческом общежитии.

### Основные возможности

- Авторизация администраторов и техников
- Управление бронированиями (просмотр, фильтры, массовая и одиночная отмена)
- CRUD оборудование (стиральные/сушильные машины, переключение статуса)
- Управление жителями общежития
- Система уведомлений о поломках
- Статистика и базовая аналитика
- Логирование действий и попыток входа

## 🛠 Технологический стек

### Backend

- **PHP 7.4+** (чистый PHP без фреймворков)
- **PostgreSQL** (через PDO)
- **Сессии** для аутентификации
- **Composer**:
  - [`phpoffice/phpspreadsheet`](https://phpspreadsheet.readthedocs.io/) – экспорт в Excel
  - [`phpoffice/phpword`](https://phpword.readthedocs.io/) – экспорт в Word

### Frontend

- HTML5 / CSS3 (адаптивный дизайн)
- JavaScript (ES6)
- **Chart.js** – построение графиков статистики
- Иконки (эмодзи / текстовые)

### Инфраструктура

- **Веб‑сервер**: Apache / Nginx
- **База данных**: Supabase (PostgreSQL) или локальный PostgreSQL
- **Docker** – для контейнеризации окружения

## Инструкция по установке и запуску

### 1. Клонирование репозитория

```bash
git clone https://github.com/Ratarion/stirka_site.git
cd stirka_site
```


### 2. Настройка веб‑сервера

* Направьте корневую директорию сервера в папку проекта (например, `public/`, если используется).
* Убедитесь, что `mod_rewrite` включён (для ЧПУ).

### 3. Настройка подключения к базе данных

Создайте файл `config/db_connect.php` (или измените существующий) с параметрами подключения к PostgreSQL:

**php**

```
<?php
$host = 'localhost';
$dbname = 'stirka_db';
$user = 'postgres';
$pass = 'your_password';

try {
    $pdo = new PDO("pgsql:host=$host;dbname=$dbname", $user, $pass);
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
} catch (PDOException $e) {
    die("DB connection failed: " . $e->getMessage());
}
```

### 4. Создание структуры БД

Выполните SQL‑скрипты из документа **«Описание структуры БД»** (файл `описанию структуры БД.docx`).
Основные таблицы: `residents`, `machines`, `booking`, `administrators`, `notifications`.

### 5. Установка зависимостей Composer

**bash**

```
composer install
```

(Если Composer не установлен глобально, используйте `php composer.phar install`)

### 6. Настройка прав доступа

Убедитесь, что веб‑сервер имеет права на запись в директории логов (если используются).

### 7. Запуск

Откройте в браузере адрес, соответствующий точке входа (например, `http://localhost/booking.php`).

### 8. Первый вход

* Используйте учётные данные администратора, предварительно добавленные в таблицу `administrators` (пароль хэшируется через `password_hash()`).
